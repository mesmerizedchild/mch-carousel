Here you will find information on how to use captions, hyperlinks and mouse-over images with MCh Carousel. If this page is your very first encounter with MCh Carousel, you might want to read the Quick Start first and take if from there [you will eventally land back onto this page...]

Non-technical: language direction and slide flow

Have you ever thought that your expectation of what happens when a carousel "moves on to the next slide", is connected to the direction in which you [usually] read?

As you read this text and your eyes scans each line, old content [i.e. the text you have already read] exits from the left edge of your field of vision, and new content [the text you yet have to read] enters from the right edge. You will expect a carousel to "move on to the next slide" in a fashion that recalls what happens when you are reading text, that is: slide old content off from the left edge of the viewport, and bring new content in from the right edge.

This is what happens to people accustomed to languages written in left-to-right scripts: the Latin script [that you are reading right now], the Greek script, Cyrillic, etc. Some languages, though, use scripts that are written and read from right to left, most notably Arabic and Hebrew. And this changes things a bit.

When people read a line of text written in the Arabic or Hebrew script and their eyes move from right to left, old content exits from the right edge of their field of vision, and new content enters from the left edge. Coherently, these same people expect that a carousel "moves on to the next slide" by sliding old content off from the right edge of the viewport, and bringing new content in from the left edge.

MCh Carousel accomodates for these different expectations.

The dir attribute

MCh Carousel is sensitive to the direction of the language of its host page, provided that the dir attribute is present somewhere in the HTML. A well-written HTML page will have this attribute at least in its html or body tag, best if together with the lang attribute.

  <!-- Sample html tag for a page written in English -->
  <html lang="en" dir="ltr">

  <!-- Sample html tag for a page written in Arabic -->
  <html lang="ar" dir="rtl">

If adding the dir attribute to the html or body tag is not possible or undesirable, it may be added to the div that starts off an instance of MCh Carousel [recall Listing the images for MCh Carousel]. For example:

  <div id="my-carousel" dir="rtl"><!-- This carousel targets a right-to-left audience -->
    <img src="images/image1.jpg" />
    <img src="images/image2.jpg" />
    ... etc. ...

Either way, MCh Carousel will detect the value of the dir attribute, and accordingly adjust the order in which the images are presented, along with the direction in which they slide automatically.

Note that, in order to display the caption line in a coherent fashion, the CSS does have to take the direction into account. The CSS for both carousels include the following:

  .mch-caption-line1 {
    position: relative;
    top: 40px;
    left: 40px;
    right: auto;
  }
  [dir="rtl"] .mch-caption-line1 {
    left: auto;
    right: 40px;
  }

This CSS snippet places the caption close to the left edge of the image for the left-to-right carousel, while placing the caption close to the right edge for the right-to-left carousel. This technique is needed especially when building themes for MCh Carousel [where the target audience is, in essence, the whole world].

Buttons

MCh Carousel has a set of built-in buttons that allow you to take control over the sideways flow of images; these buttons are always generated in the HTML code, but if CSS does not style them up, they are just hidden [as in the carousels displayed till now!]

There are 3 button panes available, each of which may be made visible or kept hidden independently:

  • Slide-control pane

    This pane comprises 2 buttons: Pause and Start. Their function is self-explanatory: if the carousel is configured to slide images automatically, the former button will pause the automatic slide, while the latter button will restart it. Clicking on Pause when the carousel is paused, or clicking on Start while the carousel is sliding images, has no effect. If the carousel is not configured to slide images automatically, neither button will have any effect either.

  • Start-end-slide pane

    This pane consists of 4 buttons: Slide to first image, Slide to next image, Slide to previous image, plus Slide to the last image. The exact direction in which each button will slide the carousel depends on the value of the dir attribute, as discussed above. This pane is useful if you want to style the buttons up with elements that change their meaning depending on the reading direction; for example, using the text "Previous" or "Next" in the buttons.

  • Left-right-slide pane

    This pane consists of 4 buttons: Slide to the leftmost image, Slide to the image to the left, Slide to the image to the right, and finally Slide to the rightmost image. The direction in which these button slide is not dependent on the value of the dir attribute. This pane is useful in case you want to style the buttons up with elements that unambiguously mean "left" and "right", for example, images of arrows pointing left or right.

Both the Start-end-slide pane and the Left-right-slide pane might need direction-dependent CSS styling, if you intend to target both audiences, but the Start-end-slide may make matters a bit easier.
