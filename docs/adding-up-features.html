MCh Carousel: Adding Up Features

Here you will find information on how to use captions, hyperlinks and mouse-over images with MCh Carousel. If this page is your very first encounter with MCh Carousel, you might want to read the Quick Start first...

All the features listed in this page are activated by adding data attributes to the <img> of interest.

Please note that, in order to display this page and other documentation pages correctly, you will need an active Internet connection, as some resources are downloaded live from the net.

Adding Captions

There are up to 3 lines of caption available for each image. In order to specify the caption you need to add data attributes as in the following example:

  <img src="images/image1.jpg"
       data-caption-line1="The first line of caption,"
       data-caption-line2="the second line of caption,"
       data-caption-line3="and the third line..." />
  <img src="images/image2.jpg"
       data-caption-line1="Some more caption, but"
       data-caption-line2="this time for the second"
       data-caption-line3="image in the carousel..." />

How these three lines are rendered depends entirely on your CSS; they may overlap the underlying image, appear below the image, above it, you name it.

And because each line can be styled independently of the others, you may implement any mix that takes your fancy: you want the first line of caption to appear above the image, the second line to overlay it, and the third line to appear to its right, rotated by 90 degrees? A bit outlandish as an idea, but totally possible by styling MCh Carousel properly!

The source code for the carousel above is in html/outlandish-carousel.html. It can also be run standalone by clicking here.

Finally, note that there is no hierarchy in the caption lines, so you may specify each and any one line that you want to display.

Freebies! ID and class

If you specify an ID or a class for your images, these will be used by MCh Carousel, and in turn your CSS may use it to change the appearance of one single image, or a group of images. For example:

  <img src="images/image1.jpg"
       id="image-1" />
  <img src="images/image2.jpg"
       class="even-image" />

Then in the associated CSS might read:

  #image-1 .mch-caption-lines {
    color: lightsalmon;
  }
  .even-image .mch-caption-lines {
    color: antiquewhite;
  }
  .mch-caption-lines {
    color: peachpuff;
  }

This way, the caption lines for the first image [with id="image-1"] would display on a light-salmon background, the second image [with class="even-image"] would display on an antique-white background, and all other images would display on a peach-puff background.

Hiding and displaying captions for individual images

A start-up option will tell the carousel when to display captions by default: always, or on mouse-over [see Options and API for more details]. However, each image can override that start-up option by adding a data attribute as in the following example:

  <!-- The caption for this image is always displayed,
         regardless of the start-up option -->
  <img src="images/image1.jpg"
       data-caption-line1="Caption always on"
       data-caption-display="always" />
  <!-- The caption for this image is displayed only 
         when hovering over it -->
  <img src="images/image2.jpg"
       data-caption-line1="Only when the mouse hovers me"
       data-caption-display="hover" />

Captions and languages

You may specify the language of each caption by adding the attribuge data-caption-lang to the <img> tag. This is useful, for example, to select different fonts for scripts other than Latin, via CSS.

Note that caption lines for any one image all share the same language, but each image may have a different language, as in the following example:

  <!-- Language of caption below is 
         English [who would have guessed!], lang="en" -->
  <img src="images/image1.jpg"
       data-caption-line1="Text in English"
       data-caption-lang="en" />
  <!-- Language of caption below is French, lang="fr" -->
  <img src="images/image2.jpg"
       data-caption-line1="Texte en français"
       data-caption-lang="fr" />
  <!-- Language of caption below is Greek, lang="el" -->
  <img src="images/image2.jpg"
       data-caption-line1="κείμενο στην ελληνική γλοσσα"
       data-caption-lang="el" />

When styling the carousel, your CSS might then include something like the following:

  .mch-caption-line1 {
    /* Source Code Pro is a very nice monospaced font,
         but it does not support the Greek script... */ 
    font-family: 'Source Code Pro' ;
  }
  [lang="el"] .mch-caption-line1 {
    /* ... whereas Roboto Mono does support it. */
    font-family: 'Roboto Mono';
  }

See Why use the language attribute? for more information on this topic.

Adding Hyperlinks

Each image may be turned into a hyperlink; that is, its whole surface may become clickable and direct you to another page. To do so, add the attribute data-link to the <img> tag, and specify there the destination of the hyperlink.

By default, the link will open in the same window where the carousel sits. You may indicate that the link should open in a new window instead, by adding the attribute data-open-in-new-window, with a value of "true".

The following example is pretty straightforward and self-documented:

  <!-- some-page.html will open in the same window as the carousel -->
  <img src="images/image1.jpg"
       data-link="some-page.html" />
  <!-- On the other hand, google.com will open in a new window -->
  <img src="images/image2.jpg"
       data-link="https://www.google.com" 
       data-open-in-new-window="true" />
  <!-- And this last image has no hyperlink at all. -->
  <img src="images/image3.jpg" />

Using mouse-over images

You may specify an alternate version of an image to be displayed when the mouse pointer hovers over it. For obvious reasons, this feature is not available on mouse-less devices...

The image that you specify for mouse-over should be the same size as the "non-mouse-over" image; MCh Carousel will behave erratically if this condition is not met [Ok, let me be honest: I have not tested it...]

  <img src="images/image1.jpg"
       data-img-over="images/image1-over.jpg" />
  <img src="images/image2.jpg"
       data-img-over="images/image2-over.jpg" />
