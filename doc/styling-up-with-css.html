MCh Carousel: Styling up with CSS

Here you will find information on how to style up your MCh Carousel, together with usable examples of styling. If this page is your very first encounter with MCh Carousel, you might want to read the Quick Start first and take if from there [you will eventally land back onto this page...]

Please note that, in order to display this page and other documentation pages correctly, you will need an active Internet connection, as some resources are downloaded live from the net.

MCh Carousel at run-time

MCh Carousel reorganises the DOM heavily, turning the simple div element with a handful of child img elements, into an articulated DOM tree [or a lush DOM forest, if you prefer...]

Understanding the DOM structure is essential to styling MCh Carousel, as it's not immediately obvious where each piece of information [that is, captions, buttons, hyperlinks, etc] ends up at run-time.

The DOM tree

The view below represents the DOM tree that MCh Carousel builds at run-time. Select each node to read a description of what it is for and whether you can style it; expand the nodes to dig into the tree.

It may look scary, but fear not, for it's not all that complicated after all...

In general, you should be wary of styling the topmost layers; think of them a bit as the scaffolding that keeps MCh Carousel together and makes it what it is. They are styled in mch-carousel.css and most of them should be left untouched.

On the other hand, you have complete control over div.mch-caption-cntnr tags and their descendants [for styling your caption text] and on the div#mch-buttons-pane tag and its descendants [for styling the button panes].

Other elements that might need your attention are highlighted in bold.

The button panes

MCh Carousel has a set of built-in buttons that allow you to take control over the sideways flow of images; these buttons are always generated in the HTML code, but if CSS does not style them up, they are just hidden [as in the carousels displayed till now!]

There are 3 button panes available, each of which may be made visible or kept hidden independently:

  • Slide-control pane

    This pane comprises 2 buttons: Pause and Start. Their function is self-explanatory: if the carousel is configured to slide images automatically, the former button will pause the automatic slide, while the latter button will restart it. Clicking on Pause when the carousel is paused, or clicking on Start while the carousel is sliding images, has no effect. If the carousel is not configured to slide images automatically, neither button will have any effect either.

  • Start-end-slide pane

    This pane consists of 4 buttons: Slide to first image, Slide to next image, Slide to previous image, plus Slide to the last image. The exact direction in which each button will slide the carousel depends on the value of the dir attribute, as discussed above. This pane is useful if you want to style the buttons up with elements that change their meaning depending on the reading direction; for example, using the text "Previous" or "Next" in the buttons.

    Displaying this navigation pane is usually mutually exclusive with displaying the left-right-slide pane.

  • Left-right-slide pane

    This pane consists of 4 buttons: Slide to the leftmost image, Slide to the image to the left, Slide to the image to the right, and finally Slide to the rightmost image. The direction in which these button slide is not dependent on the value of the dir attribute. This pane is useful in case you want to style the buttons up with elements that unambiguously mean "left" and "right", for example, images of arrows pointing left or right.

    Displaying this navigation pane is usually mutually exclusive with displaying the start-end-slide pane.

Both the Start-end-slide pane and the Left-right-slide pane might need direction-dependent CSS styling, if you intend to target both audiences, but the Start-end-slide may make matters a bit easier.
