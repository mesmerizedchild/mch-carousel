MCh Carousel: Adding Up Features

Here you will find information on how to use captions, hyperlinks, buttons and mouse-over images with MCh Carousel. If this page is your very first encounter with MCh Carousel, you might want to read the Quick Start first...

All the features listed in this page are activated by adding data attributes to the <img> of interest.

Please note that, in order to display this page and other documentation pages correctly, you will need an active Internet connection, as some resources are downloaded live from the net.

Adding Captions

There are up to 3 lines of caption available for each image. In order to specify the caption you need to add data attributes as in the following example:

  <img src="images/image1.jpg"
       data-caption-line1="The first line of caption,"
       data-caption-line2="the second line of caption,"
       data-caption-line3="and the third line..." />
  <img src="images/image2.jpg"
       data-caption-line1="Some more caption, but"
       data-caption-line2="this time for the second"
       data-caption-line3="image in the carousel..." />

How these three lines are rendered depends entirely on your CSS. The page Styling up with CSS contains more information about this subject, but below you'll find some built-in CSS files to get you started...

Note that each caption line can be styled independently of the others, so you may implement any style that takes your fancy: you want the first line of caption to appear above the image, the second line to overlay it, and the third line to appear to its right, rotated by 90 degrees?   <link rel="stylesheet" href="css/mch-carousel-outlandish-caption.css">

For a more-sedate caption style, you can use:

  <link rel="stylesheet" href="css/mch-carousel-coloured-caption.css">

... or if you want to go for plain boring:

  <link rel="stylesheet" href="css/mch-carousel-plain-caption.css">

Finally, there is no hierarchy in the caption lines, so you may specify each and any one line that you want to display. You could use this feature to have only one line of caption for each image, but be able to use 3 different styles without too much CSS trickery.

Adding Hyperlinks

Each image may be turned into a hyperlink; that is, its whole surface may become clickable and direct you to another page. To do so, add the attribute data-link to the <img> tag, and specify there the destination of the hyperlink.

By default, the link will open in the same window where the carousel sits. You may indicate that the link should open in a new window instead, by adding the attribute data-open-in-new-window, with a value of "true".

The following example is pretty straightforward and self-documented:

  <!-- some-page.html will open in the same window as the carousel -->
  <img src="images/image1.jpg"
       data-link="some-page.html" />
  <!-- On the other hand, google.com will open in a new window -->
  <img src="images/image2.jpg"
       data-link="https://www.google.com" 
       data-open-in-new-window="true" />
  <!-- And this last image has no hyperlink at all. -->
  <img src="images/image3.jpg" />

Using mouse-over images

You may specify an alternate version of an image to be displayed when the mouse pointer hovers over it. For obvious reasons, this feature is not available on mouse-less devices...

The image that you specify for mouse-over should be the same size as the "non-mouse-over" image; MCh Carousel will behave erratically if this condition is not met [Ok, let me be honest: I've tested it, the result is messy, and I'm not sure that fixing this is a priority at the moment...]

  <img src="images/image1.jpg"
       data-img-over="images/image1-over.jpg" />
  <img src="images/image2.jpg"
       data-img-over="images/image2-over.jpg" />

Button panes

MCh Carousel also supports navigation and slide-control buttons, so that you can slide images back and forth at the click of a button, or pause/restart the automatic slide. These buttons are styled entirely in CSS; if left non-styled, as in the previous carousels, these buttons are not visible.

Just as with caption text, a better understanding of the run-time DOM structure of MCh Carousel is necessary before you will be able to style the buttons up according to your preferences [and, as before, you may refer to Styling up with CSS]. For the time being, you can reuse one of the CSS files included in the documentation. For dodge blue buttons:

  <link rel="stylesheet" href="css/mch-carousel-dodgeblue-buttons.css">

Last but not least: ID and class

If you specify an ID or a class for your images, these will be used up by MCh Carousel, and your CSS can be written accordingly. For example:

  <img src="images/image1.jpg"
       id="image-1" />
  <img src="images/image2.jpg"
       class="even-image" />
  <img src="images/image3.jpg"
       id="image-3" class="odd-image" />

Note that these ID and/or class will be "moved" to other elements at run-time; your CSS, however, can still rely on these ID's and classes to change the appearance of one single image, or a group of images.

Just to recap, the features that this carousel portraits are:

  • Automatic sliding:

    When the images do not fit in the viewport of the carousel, it will start to slide them automatically; this feature may be switched off.

  • Support for the mousewheel, touchscreen and trackpads:

    If you hover your mouse pointer over the carousel, or touch it with your finger, you may use any of these methods to take control of scrolling; while the mouse hovers over the carousel, or while your finger touches it, the automatic slide is disabled.

  • Caption lines:

    It reinforces the idea that each line may be styled independently. Also note the round-corner border around the semi-transparent background [the caption text is fully opaque]. You might want to be less flashy in a real-life carousel, but you have the choice...

  • Hyperlinks:

    All images may be clicked, and a new page will open; for all images the hyperlink will open in a new window [or a new tab], except for the fifth image [the starfish], whereby the page will open where the carousel sits.

  • Mouse-over images:

    All images except the last one have a mouse-over image. No big mystery here...

  • Buttons:

    This theme has styled up the navigation and slide-control panes. The fact that the buttons are below the viewport is a stylistic choice; if preferred, the buttons could overlay the viewport, with some transparency effect if you so wish.

  • ID and Class:

    The third and the fourth images declare a class in the <img> tag; MCh Carousel picks it up, and the CSS in turn uses it to change the appearance of the caption text. The third image has an additional class, which changes the colour of the caption to plain black. This carousel does not use ID on any images, but you get the idea of what is possible... The CSS used in the carousel is the aforementioned css/mch-carousel-coloured-caption.css
