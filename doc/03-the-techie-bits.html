MCh Carousel: Integrating into pages

This page describes how to listen to events triggered by MCh Carousel, how to programmatically slide pages, and the structure of the DOM tree so that you can customise your CSS to meet your needs. If this page is your very first encounter with MCh Carousel, you might want to read the Quick Start first and take if from there [you will eventally land back onto this page...]

Please note that, in order to display this page and other documentation pages correctly, you will need an active Internet connection, as some resources are downloaded live from the net.

The full list of events may be obtained via:

  <script type="text/javascript">
    alert($.fn.mchCarousel.eventNames);
  </script>

Î•vent names all start with mch-carousel:, followed by a self-descriptive string, telling what the event is about.

When you subscribe to an MCh Carousel event, the attribute target of the incoming event references the element within MCh Carousel that originated the event, as you would expect. The possible classes of event sources are:

  • The carousel itself [enter and leave events].
  • The sliding viewport [slide-related events].
  • Any one of the images [enter, leave, click and auto-centre events].
  • Any one of the buttons [enter, leave and click events].

Returning false from an event handler, or invoking stopPropagation() on the event object, will not have any effect on the carousel.

Check the source code in src/mch-carousel.js, where it reads: /* Public event names */, for more details.

The DOM tree and the CSS

MCh Carousel reorganises the DOM heavily; starting from the simple div element with a handful of child img elements that you define in your HTML, it generates an articulated DOM tree [or a lush DOM forest, if you prefer...]

Understanding the DOM structure is essential to styling MCh Carousel, as it's not immediately obvious where each piece of information [that is, captions, buttons, hyperlinks, etc] ends up at run-time.

The view below represents the DOM tree that MCh Carousel builds at run-time. Select each node to read a description of what it is for and whether you can style it; expand the nodes to dig into the tree.

It may look scary, but fear not, for it's not all that complicated after all...

In general, you should be wary of styling the topmost layers; think of them a bit as the scaffolding that keeps MCh Carousel together and makes it what it is. They are styled in mch-carousel.css and most of them should be left untouched.

On the other hand, you have complete control over div.mch-caption-cntnr tags and their descendants [for styling your caption text] and on the div#mch-buttons-pane tag and its descendants [for styling the button panes].

Other elements that might need your attention are highlighted in bold.

The button panes

MCh Carousel has a set of built-in buttons that allow you to take control over the sideways flow of images; these buttons are always generated in the HTML code, but if CSS does not style them up, they are just hidden [as in the carousels displayed till now!]

There are 3 button panes available, each of which may be made visible or kept hidden independently:

  • Slide-control pane

    This pane comprises 2 buttons, quite easy to understand:

    Button name

    ID

    Function

    Pause

    		 #mch-pause

    This button will pause the automatic sliding. Clicking on Pause when the carousel is already paused, has no effect.

    Start

    		 #mch-restart This button will [re]start the automatic sliding. Clicking on Start when the automatic sliding is already running, has no effect.

  • Left-right-slide pane

    This pane consists of 4 buttons; their sliding direction does not depend on the language direction in the current page.

    Sliding direction

    ID

    Function

    Leftmost image

    #mch-leftmost

    Slide the carousel to the leftmost image, regardless of the language direction of the current page.

    Left

    #mch-left

    Slide the carousel to the left, regardless of the language direction of the current page.

    Right

    #mch-right

    Slide the carousel to the right, regardless of the language direction of the current page.

    Rightmost image

    #mch-rightmost

    Slide the carousel to the rightmost image, regardless of the language direction of the current page.

    This pane is useful in case you want to style the buttons up with elements that unambiguously mean "left" and "right", for example, images of arrows pointing left or right.

    Displaying this navigation pane is usually mutually exclusive with displaying the start-end-slide pane [below], but nothing in MCh Carousel enforces this principle.

  • Start-end-slide pane

    This pane consists of 4 buttons; their sliding direction does depend on the language direction in the current page, so the physical outcome of clicking on each of these buttons will depend on whether the current language is left-to-right [LTR for short: English, Spanish, German, etc.] or right-to-left [RTL for short: mostly Arabic and Hebrew]; see Last but not least: BiDi for more details.

    Sliding direction

    ID

    Function

    First image

    mch-first

    Slide the carousel to the first image; this will translate to "the leftmost image" for LTR languages, and to "the rightmost image" for RTL languages.

    Previous image

    #mch-prev

    Slide the carousel to the previous image; this will translate to "the image to the left" for LTR languages, and to "the image to the right" for RTL languages.

    Next image

    #mch-next

    Slide the carousel to the next image; this will translate to "the image to the right" for LTR languages, and to "the image to the left" for RTL languages.

    Last image

    #mch-last

    Slide the carousel to the last image; this will translate to "the rightmost image" for LTR languages, and to "the leftmost image" for RTL languages.

    Displaying this navigation pane is usually mutually exclusive with displaying the left-right-slide pane [above], but nothing in MCh Carousel enforces this principle.

BiDi II, The Return

The section Last but not least: BiDi explains that MCh Carousel is sensitive to the language direction of its host page [best by specifying the attribute dir somewhere in the HTML].

Be aware, though, that CSS really is not BiDi-aware; most of the times CSS has attributes such as margin-left or margin-right, but not margin-start nor margin-end; better said, some browsers may support such options via browser-specific prefixes [for example -moz-margin-start for Firefox], but these are not part of any CSS standard yet, and as such cannot be relied upon for cross-browser, BiDi-aware development.

The net effect is that if you want something in the carousel to appear "close to the start" or "close to the end", then you will need to specify both options explicitly in your CSS, with a selector such as [dir="rtl"]:

  /* Caption line aligned to the "start" of the image: */
  .mch-caption-line1 {
    position: absolute;
    top: 0;
    left: 0;
    /* line-height: 125px; etc... */
  }
  [dir="rtl"] .mch-caption-line1 {
    /* For RTL languages, reset 'left' and set 'right'. */
    left: auto;
    right: 0;
  }

  /* Similarly, for a button aligned to the "end" of the viewport: */
  #mch-last {
    position: absolute;
    bottom: 0;
    right: 0;
    /* line-height: 32px; etc... */
  }
  [dir="rtl"] #mch-last {
    /* For RTL languages, reset 'right' and set 'left'. */
    left: 0;
    right: auto;
  }
